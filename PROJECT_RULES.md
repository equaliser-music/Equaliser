# Project Rules

## Folder Separation

### `mockups/`
- **Purpose**: UX exploration and prototyping
- **Approach**: Leave files as-is; used for visual design and basic mock functionality to get a feel for things
- **Do not**: Modify existing mockup files unless specifically requested

### `content_node/`
- **Purpose**: Real, working functionality
- **Approach**: Build production-ready code as described in specification documents
- **Standards**: Follow Technical Specification and Functional Specification documents

## Key Principle

Treat `content_node/` tasks as completely separate from `mockups/`. They serve different purposes and should not be conflated.
